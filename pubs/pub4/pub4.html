Researches about theory (R)

1_R) Describe the notion of statistical population. What is the population in descriptive statistics and what is the population in inferential statistics: point out the differences.

A statistical population is a set of items/events involved in a statistical exam, we can refer at it as all the data we're interested in. The descriptive statistics, as the name tells us, tries to explain in a better way the data and formats them in a more readable and comprehensive (descriptive of course) way. So, the statistical population in descriptive statistics is all the raw data we would like to understand, present in a readable way or just point out some interesting correlations or pattern between data. We cannot make conclusions on the population, it only describes in a meaningful way data. The inferential statistics instead allow us to figure out some conclusions on the data inputted. Sometimes we can't analyze all the population (e.g. I would like to mark all the university students in the world), so we can just sample the data. The sampling method extrapolates groups of items in the population that can represent all the data. The statistical population in the inferential statistics refers to only data we really process to make conclusions.

2_R) Describe the notion of statistical attributes/variables and dataset, and explain how a dataset is generated.

A statistical attribute is a characteristic of an entity, e.g. age or nationality of a person. These attributes can take some kind of values, and all the values contitute a domain. A domain is a range of values, and the attribute could change value as some events happens/change. A variable is the objectification of an attribute, so can vary during the statistical analysis. A dataset is a collection of data. The informations involved can be quantitative or qualitative, or they can even miss. In the case of a tabular dataset, we have some columns and rows. The columns define the variables involved for a specific attribute and the rows define the values for a specific entity for all the attributes. This is an example of a dataset.

Company Revenue (2019) Employees Country
Google 160.74 bln $ 115'000 USA
Ferrari 3.767 bln EUR 4'285 Italy
GigaByte 1.94 bln $ - Taiwan
IBM 77.14 bln $ 352'600 USA
In this case we have a dataset involving 4 attributes. We have the company name, the revenue of the 2019, the number of employees and the state of the headquarter. There are 4 rows here, so our dataset describes 4 entity. All the data involved in the records are variables, because they are a particular instance of an attributes, for a particular entity; and these values could change as the time pass.

2_A) Create or search some simple but illuminating example of code which clearly shows the different behaviors of reference value data types and value type data types.

The Types in .NET Framework, but this is applied for all programming languages having concepts of pointers, are either treated by Value Type or by Reference Type.

"A Value Type holds the data within its own memory allocation and a Reference Type contains a pointer to another memory location that holds the real data. Reference Type variables are stored in the heap while Value Type variables are stored in the stack" [2].
Code Example: Passing Value Type Variables 
static void ChangeValue(int x)
{
    x =  200;
    Console.WriteLine(x);
}

static void Main(string[] args)
{
    int i = 100;

    Console.WriteLine(i);
    
    ChangeValue(i);
    
    Console.WriteLine(i);
}

// OUTPUT:
100
200
100
Code Example: Passing Reference Type Variable 
static void ChangeReferenceType(Student std2)
{
    std2.StudentName = "Steve";
}
                                
static void Main(string[] args)
{
    Student std1 = new Student();
    std1.StudentName = "Bill";
                                    
    ChangeReferenceType(std1);
                                
    Console.WriteLine(std1.StudentName);
}

// OUTPUT:
Steve
